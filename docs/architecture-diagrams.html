C4 Level 1 - System Context

Overview

SUSE AI Factory is the AI platform management layer built into Rancher. It provides a unified interface to discover AI applications, compose them into validated stacks, and deploy AI workloads across Kubernetes clusters.

🏗️ Single Binary, Split Concerns

The operator is a single Go binary with clean separation: cmd/operator/main.go (≤250 lines) handles wiring, internal/api has HTTP handlers, internal/controller has reconcilers, and pkg has business logic.

Key Architectural Decisions

1. Kubernetes-Native

All state stored as CRs. No external database. In-memory caches rebuild from CRD state on startup.

2. Least Privilege

Fine-grained RBAC. No cluster-admin. Every resource explicitly listed.

3. Secrets Management

Credentials in K8s Secrets only. Consumed via secretKeyRef. Never in values.yaml.

4. Blueprint Immutability

Spec fields immutable per version. Webhook-enforced. New versions = new CR instances.

5. Workload Provenance

Every Workload records spec.source (App/Blueprint/BundleTest).

6. Air-Gap First-Class

No direct NVIDIA NGC access. All assets via SUSE Registry mirror.

Critical Constraints

⚠️ Non-Negotiable Architecture Constraints

  • No internal OCI registry — AIF does not host/proxy/mirror images
  • No direct NVIDIA NGC access — All assets via SUSE Registry mirror
  • Blueprint immutability — Spec fields immutable (webhook-enforced)
  • Air-gap parameterization — Registry endpoints configurable via Settings
  • Publish-by-approval governance — Bundle → Submitted → Approved → Blueprint
  • Single pull-secret pattern — One docker-config Secret per workload namespace
  • Hub-on-management — Operator runs ONLY on Rancher management cluster
